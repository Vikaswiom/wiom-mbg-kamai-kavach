The MBG guarantee rewards a partner for installing 60%+ of their leads. A partner's efficiency = leads installed ÷ leads that reached an end state (a slot-confirmed lead that finished — installed or terminally cancelled; still-open leads aren't counted yet). 60%+ = "secured", cleared the gate — the 60%+/− split is measured against partners with at least one resolved lead (efficiency is undefined without leads). Method & definitions
  • Segments: A & B split the 425 MBG-eligible cspids by whether they fired a banner_opened event (CLEVERTAP_CSP_API.EVENTS_DATA, mapped via PROFILE_DATA) since the window start. C = Non-MBG CSPs that were active in the install pipeline (base = active only, since the full Non-MBG population is unbounded).
  • Funnel stages (per CSP, ≥1) from DBT_CSP.TAS_INSTALL_EXECUTION_CANDIDATES (ETL_CURRENT=TRUE): Slot = CONFIRMED_SLOT_AT · Technician = EXECUTOR_ID · Installed = OTP_VERIFIED / COMPLETED_STEP≥7, joined on CSP_ID.
  • Scope: installs are measured on the new F1 OS CSP app pipeline (TAS_INSTALL_EXECUTION_CANDIDATES) — the app the MBG banner lives in. All three segments are measured identically within the same July window, so the comparison is like-for-like.
  • Maturity: installs resolve fast — ~97% land within 3 days of task creation — so this ~4-day window is largely mature, but its newest tasks are still in flight, so install rates are a floor that will edge up.
  • Partner efficiency & 60% gate replicate the live MBG "Kamai Kavach" calc (same numbers the CSP-facing banner shows). From raw CSP_TAS_SERVICE.INSTALL_EXECUTION_CANDIDATES (_FIVETRAN_ACTIVE=TRUE), one row per (CONNECTION_ID, CSP_ID) with final state via MAX_BY(_, UPDATED_AT): leads (denom) = slot-confirmed leads (CONFIRMED_SLOT_AT) that reached an end state that month (excludes still-open & system cancels); installs = those that finished installed; efficiency = installs/leads; 60%+ ("secured") = installs/leads > 0.60. Month = the month the lead reached its end state. MBG launched 1 July → June is the pre-MBG baseline for the same partners. Enrolled = Flow-1 "ENROLLED" / audit done in the belief-break onboarding audit (429 partners); eligible = offered but not enrolled — Flow-1 not-enrolled + all Flow-2 (140); Non-MBG = everyone else. partner_id→csp_id via CSP_GATEWAY_SERVICE.CSP_ACCOUNT. July matches wiom-mbg-kamai-kavach/data.json.
  • A vs B is the clean test — both are full sub-populations of the same 425 cohort, so the difference isolates banner engagement. C is a labeled network benchmark. Association, not proof of cause — CSPs who click are likely more engaged to begin with. Auto-refreshes hourly.
